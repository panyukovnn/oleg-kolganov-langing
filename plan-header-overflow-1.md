# План: шапка уходит вправо за пределы контейнера

## Проблема
`.kn-header__row` (main-style.css:158) — flex-строка. Все её дети не сжимаются:
`.kn-logo` (`flex-shrink:0`, :168), `.kn-meta-item` ×4 (`white-space:nowrap` + `flex-shrink:0`, :190-198),
`.kn-header__cta` (:224). При этом `.kn-header__meta` прячется только при `≤1024px` (:1704),
а контейнер `--container-w: 1240px` (:58). В диапазоне **~1025–1290px** суммарная
минимальная ширина детей больше контейнера → строка вылезает вправо (обрезается `overflow-x: clip`).

## Варианты решения (выбрать один)

### Вариант A (рекомендуемый) — поднять брейкпоинт скрытия meta
- [ ] В media `@media (max-width: 1024px)` убрать строку `.kn-header__meta { display: none; }` (:1704)
- [ ] Добавить отдельный media `@media (max-width: 1300px) { .kn-header__meta { display: none; } }` (порог чуть выше `--container-w`, чтобы meta показывалась только когда реально влезает)
- [ ] Проверить, что при 1025–1300px шапка = лого слева + CTA справа, без переполнения
- Плюсы: минимальная правка, гарантированно убирает overflow. Минусы: meta-блок исчезает раньше.

### Вариант B — разрешить сжатие/перенос meta
- [ ] В `.kn-meta-item` (:190) заменить `flex-shrink: 0` на `min-width: 0` (разрешить сжатие)
- [ ] В `.kn-header__meta` (:182) добавить `min-width: 0` и `overflow: hidden`
- [ ] У `.kn-meta-item__value` добавить `overflow: hidden; text-overflow: ellipsis` для усечения длинного адреса
- [ ] Проверить, что текст не «прыгает» и читается на 1025–1240px
- Плюсы: meta остаётся видимой. Минусы: адрес может усекаться многоточием.

### Вариант C — уменьшить интринсик-ширину строки
- [ ] Уменьшить `.kn-header__row { gap: 28px }` → `gap: 16px` (:161)
- [ ] Уменьшить `.kn-header__meta { gap: 24px }` → `gap: 14px` (:186)
- [ ] При необходимости уменьшить `font-size` у `.kn-meta-item` (13px → 12px, :194)
- [ ] Проверить, влезает ли строка в контейнер во всём диапазоне 1025–1240px
- Плюсы: ничего не прячется. Минусы: может не хватить запаса на пограничных ширинах.

### Вариант D — прогрессивно прятать часть meta-пунктов
- [ ] Оставить в diапазоне 1025–1300px только «Адрес» + «Режим работы», скрыв «Метро» и «Эл. почта» через media-правило по их позиции/классу
- [ ] (требует добавить классы-модификаторы пунктам в _includes/header.html, напр. `kn-meta-item--secondary`)
- [ ] Проверить отсутствие переполнения
- Плюсы: компромисс между A и B. Минусы: правка и в HTML, и в CSS.

## Проверка (для любого варианта)
- [ ] Прогнать ширины окна: 1280, 1200, 1100, 1025, 1024, 768, 480, 400 — шапка не выходит за `.kn-container`
- [ ] Проверить, что автоскрытие шапки (`.kn-header--hidden`, header-autohide.js) и sticky не сломаны
- [ ] Пересобрать Jekyll (`bundle exec jekyll build` в `docs/`) и проверить главную + внутреннюю страницу
